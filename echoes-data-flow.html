Step-by-step, with the actual payloads

1
Frontend · Mapbox GL JS
User taps a building on the Brooklyn map
Mapbox fires a click event on the map. You grab the lng/lat from the event, and optionally the nearest feature (building footprint, street name) from queryRenderedFeatures.
// In your React map component map.on('click', async (e) => { const { lng, lat } = e.lngLat; const features = map.queryRenderedFeatures(e.point); const hit = features[0]; // nearest building / poi if any const res = await fetch('/api/story', { method: 'POST', body: JSON.stringify({ lng, lat, featureName: hit?.properties?.name ?? null, layer: hit?.layer?.id ?? null }) }); const { story, audioUrl, sources } = await res.json(); playAudio(audioUrl); showCard(story, sources); });
▼ HTTP POST /api/story ▼
2
Next.js API routeMapbox Geocoding
Turn lat/lng into a searchable place (reverse geocode)
Tavily searches text, not coordinates. Convert the tap point into "1 Washington Ave, Fort Greene, Brooklyn" using Mapbox's Geocoding API so Tavily has something to search for.
// /app/api/story/route.ts const geo = await fetch( `https://api.mapbox.com/geocoding/v5/mapbox.places/${lng},${lat}.json` + `?access_token=${process.env.MAPBOX_TOKEN}&types=address,neighborhood` ).then(r => r.json()); const place = geo.features[0]; // e.g. "63 Flushing Ave, Brooklyn" const neighborhood = place.context.find(c => c.id.startsWith('neighborhood'))?.text; const query = `History of ${place.place_name}, ${neighborhood}, Brooklyn NY — ` + `landmark records, news archives, census, notable events`;
▼ server-side fetch ▼
3
Tavily API
Pull verified historical facts for that address
Use Tavily's /search endpoint with search_depth: "advanced" and include_raw_content: true. Tavily returns ranked results from news archives, landmark DBs, public records.
const tavily = await fetch('https://api.tavily.com/search', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ api_key: process.env.TAVILY_API_KEY, query, search_depth: 'advanced', include_answer: true, include_raw_content: true, max_results: 6, include_domains: [ 'nyc.gov', 'nypl.org', 'brooklynhistory.org', 'nytimes.com', 'brooklyneagle.com', 'wikipedia.org' ] }) }).then(r => r.json()); // tavily.answer → one-paragraph synthesis // tavily.results[] → { title, url, content, raw_content }
Fallback: if results is thin, re-query at the block level, then the neighborhood level, then the era. Something always surfaces.
▼ pass facts into Claude ▼
4
Claude API
Turn raw facts into a 45–60s first-person narrative
Claude gets the Tavily results and returns structured JSON: the story, plus metadata you need to pick a voice (era, gender, accent, emotional tone).
const claude = await anthropic.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 1200, system: `You are Echoes. Given verified historical facts about a Brooklyn location, write a 45–60 second first-person monologue from a FICTIONAL but plausible narrator of that era. Never invent facts beyond what the sources support. Return strict JSON.`, messages: [{ role: 'user', content: `LOCATION: ${place.place_name} SOURCES:\n${tavily.results.map(r => `- ${r.title}: ${r.content}`).join('\n')} Return JSON with this shape: { "story": "", "narrator": { "era": "1940s", "gender": "female", "age": 34, "accent": "brooklyn-working-class", "tone": "nostalgic" }, "sources": [{ "title": "...", "url": "..." }] }` }] }); const { story, narrator, sources } = JSON.parse(claude.content[0].text);
▼ story + narrator profile ▼
5
ElevenLabs API
Generate the voice and stream MP3 back
Map Claude's narrator profile to an ElevenLabs voice_id (pre-pick 6–10 voices covering eras/genders/accents). Hit /text-to-speech/{voice_id}/stream and get MP3 bytes.
const voiceId = pickVoice(narrator); // small lookup table you maintain const audio = await fetch( `https://api.elevenlabs.io/v1/text-to-speech/${voiceId}/stream`, { method: 'POST', headers: { 'xi-api-key': process.env.ELEVENLABS_KEY, 'Content-Type': 'application/json', 'Accept': 'audio/mpeg' }, body: JSON.stringify({ text: story, model_id: 'eleven_turbo_v2_5', voice_settings: { stability: 0.45, similarity_boost: 0.75, style: 0.35 } }) }); // Option A (simplest): buffer the MP3, upload to Supabase Storage, // return a public URL. // Option B (faster first-audio): stream bytes straight through your // API route as a ReadableStream to the browser. const buf = Buffer.from(await audio.arrayBuffer()); const { data } = await supabase.storage.from('echoes') .upload(`${crypto.randomUUID()}.mp3`, buf, { contentType: 'audio/mpeg' }); const audioUrl = supabase.storage.from('echoes').getPublicUrl(data.path).data.publicUrl;
▼ JSON response ▼
6
API routeBrowser
Return everything to the page, play + animate
Your API responds with one JSON blob. The frontend plays the MP3, drives the waveform from the audio element, and flips up the fact card with Tavily's sources.
// API response shape { "audioUrl": "https://.../echoes/abc123.mp3", "story": "I was seventeen the summer the Navy Yard hired me…", "narrator": { "era": "1940s", "tone": "nostalgic", "accent": "brooklyn-working-class" }, "sources": [{ "title": "Brooklyn Navy Yard, 1943", "url": "https://..." }] } // Client const audioEl = new Audio(audioUrl); const ctx = new AudioContext(); const src = ctx.createMediaElementSource(audioEl); const analyser = ctx.createAnalyser(); src.connect(analyser); analyser.connect(ctx.destination); audioEl.play(); requestAnimationFrame(function draw() { analyser.getByteFrequencyData(dataArray); drawWaveform(dataArray); // pulsing waveform on the map requestAnimationFrame(draw); });
Two things worth deciding early:
• Streaming vs. buffering the audio. Buffer-then-URL (Option A) is dead simple and lets Supabase cache it so the same tap is free next time. Streaming (Option B) cuts time-to-first-sound by ~1–2s but is trickier with Next.js route handlers — worth it only if judges will notice the delay.
• Cache by rounded coordinates. Round lng/lat to ~4 decimal places and key your Supabase cache on that. Same corner = same story = zero API cost on re-tap. Huge for demo day where you'll tap the same blocks repeatedly.
